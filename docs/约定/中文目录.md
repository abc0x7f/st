# 中文目录结构建议

```text
project/
├─ README.md
├─ requirements.txt
├─ .gitignore
├─ data/
│  ├─ 原始数据/                  # 原始数据，只进不改
│  ├─ 外部资料/                  # 外部底图、geojson、公开补充资料
│  ├─ 中间数据/                  # 清洗后但未建模的中间数据
│  └─ 最终数据/                  # 最终建模数据、面板数据、矩阵数据
├─ xxx/(放置代码，结构和output一样，多一个通用文件夹)
├─ output/(放置代码输出结果)
│  ├─ 数据清洗/                  # 数据清洗、合并、构造指标
│  ├─ 效率测算/                  # SBM、超效率SBM、Malmquist
│  ├─ 回归/                  # 基准回归、稳健性、异质性、诊断
│  └─ 空间计量/                  # Moran's I、LISA、空间矩阵
├─ xxx/
│  ├─ 论文/[子目录：图、表、日志]
│  ├─ 比赛/[子目录：图、表、日志]
│  └─ 备份/
├─ docs/
│  ├─ 对话记录/                  # memory*.md、过程记录、讨论稿
│  ├─ 概念解释/
│  ├─ 约定/                
│  ├─ 参考资料/                  
│  └─ 待解决问题/                
├─ presentation/
│  ├─ 提交材料/                  # 比赛最终提交材料
│  ├─ 视频材料/                  # 分镜脚本、旁白稿、素材清单
│  ├─ 演示文稿/                  # PPT
│  └─ 摘要说明/                  # 摘要、作品说明
└─ archive/
   ├─ 旧版文件/                  # 旧稿、废弃方案
   ├─ 压缩包/                    # 压缩包
   └─ 临时文件/                  # 临时文件，不进入正式流程
```

## 使用说明

### `data/`
- `原始数据/`：外部获取后不再手动修改的数据。
- `外部资料/`：地图底图、行政区划、标准系数表、公开参考文件。
- `中间数据/`：清洗、转换、拼接后的阶段性数据。
- `最终数据/`：可直接进入效率测算、回归分析、空间分析的数据。

### `src/`
- `数据处理/`：原始数据整理、字段统一、面板构造。
- `效率测算/`：DEA、SBM、超效率 SBM、Malmquist 相关脚本。
- `回归分析/`：基准回归、稳健性、异质性、模型诊断。
- `空间分析/`：空间权重矩阵、Moran's I、LISA、空间图件。
- `通用工具/`：公共函数、路径配置、字体配置、配色配置。

### `outputs/`
- `图件/论文版/`：用于论文或正式写作的图。
- `图件/比赛版/`：用于比赛展示、视频、PPT 的图。
- `图件/备份/`：历史图件或未最终采用版本。
- `表格/`：结果表、统计表、检验表。
- `模型输出/`：模型中间结果、日志、参数文件。
- `地图/`：地图类图件，可单独集中管理。

### `docs/`
- `过程记录/`：研究笔记、memory、讨论记录。
- `方法说明/`：变量定义、指标口径、方法说明。
- `分析报告/`：阶段性分析结论、结果解读。
- `参考资料/`：数据来源、参考文献、补充说明。

### `presentation/`
- `提交材料/`：比赛最终提交包。
- `视频材料/`：视频脚本、旁白稿、素材清单、成片相关文件。
- `演示文稿/`：PPT 及其配套素材。
- `摘要说明/`：摘要、作品简介、项目说明。

### `archive/`
- `旧版文件/`：旧版方案、弃用稿件。
- `压缩包/`：历史 zip、打包文件。
- `临时文件/`：临时导出、测试残留文件。

## 命名建议

- 新增脚本文件名建议保留英文，便于运行和维护。
- 目录名可以中文化，便于你自己快速识别。
- 比赛展示相关内容优先放到 `presentation/` 下统一管理。
- 最终正式分析所用数据，统一落到 `data/最终数据/`。
