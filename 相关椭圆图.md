## 三、数据组织要求

下次如果要复用，数据至少需要满足以下条件：

### 1. 数据结构

最好是一个 `csv` 或 `xlsx` 表，按“行是观测，列是变量”的形式组织。

例如：

| province | year | eff | lntl | ind | urb | rd | open | es |
|:---|---:|---:|---:|---:|---:|---:|---:|---:|
| 上海 | 2015 | ... | ... | ... | ... | ... | ... | ... |
| 云南 | 2015 | ... | ... | ... | ... | ... | ... | ... |

### 2. 变量类型

参与相关性计算的字段必须是数值型，或至少能安全转换成数值型。

### 3. 缺失值处理

当前脚本口径是：

- 对用于相关性和 `VIF` 的变量列做数值化转换
- 然后删除这些变量中含缺失值的观测行

即：

```python
for col in CORR_VARS:
    df[col] = pd.to_numeric(df[col], errors="coerce")

df = df.dropna(subset=CORR_VARS).copy()
```

因此，如果新数据缺失很多，最终样本量会变少，图和结果会随之变化。

## 五、Pearson 与 Spearman 的计算口径

### 1. Pearson

`Pearson` 用于衡量线性相关关系。

代码口径：

```python
coef, pvalue = pearsonr(x, y)
```

### 2. Spearman

`Spearman` 用于衡量秩相关或单调相关关系。

代码口径：

```python
coef, pvalue = spearmanr(x, y)
```

### 3. 当前图中如何使用

虽然代码也计算了 `pvalue`，但**图中最终不显示显著性星号**，仅保存为附表。

也就是说：

- 图主要展示相关方向和大小
- `p` 值只保留在 `csv` 中供参考

## 六、椭圆图绘制逻辑

### 1. 整体结构

每个变量两两配对，形成一个相关系数矩阵。

对矩阵中每个位置：

- 若在下三角：画椭圆
- 若在上三角：写数值
- 若在对角线：写变量名

### 2. 下三角椭圆

椭圆的方向和扁平程度由相关系数决定。

当前使用的逻辑是：

```python
abs_value = abs(value)
scale = 0.9
width = scale * np.sqrt(1 + abs_value) / np.sqrt(2)
height = scale * np.sqrt(1 - abs_value) / np.sqrt(2)
angle = 45 if value >= 0 else -45
```

解释如下：

- `r` 越接近 `0`，椭圆越接近圆形
- `r` 越接近 `1`，椭圆越扁，向右上倾斜
- `r` 越接近 `-1`，椭圆越扁，向左上倾斜

### 3. 为什么不用旧的激进压扁方案

之前曾用过更激进的纵横比映射，会让中等相关看起来像强相关。

当前方案是修正后的稳定版本，后续不要再退回旧方案。

### 4. 椭圆边框

当前固定为：

```python
edgecolor="none"
linewidth=0
```

即椭圆没有黑边框。

## 七、上三角数值显示逻辑

### 1. 数值内容

当前上三角显示：

```python
f"{value:.2f}"
```

即保留两位小数。

### 2. 不显示星号

当前不显示：

- `*`
- `**`
- `***`

因为本项目已经明确：

- 大样本面板数据下很多相关都可能显著
- 星号容易被误读为“强相关”

因此最终图中不放显著性标记。

### 3. 文字颜色限制

当前文字颜色仍跟随 `RdBu_r` 色带，但增加了一个限制：

- 不允许文字颜色落在色带最中间那段最浅区域

当前代码为：

```python
def text_color_for_value(value, cmap, norm):
    display_value = value
    center_gap = 0.18
    if 0 <= display_value < center_gap:
        display_value = center_gap
    elif -center_gap < display_value < 0:
        display_value = -center_gap
    return cmap(norm(display_value))
```

它的含义是：

- 若相关系数非常接近 `0`，文字颜色仍保留正负方向
- 但会避开色带中心那块接近白色的位置
- 这样文字不会发虚看不清

这是当前图形的关键细节，后续复用时应保留。

## 八、配色与色条

### 1. 当前色带

当前固定使用：

```python
cmap = plt.get_cmap("RdBu_r")
norm = colors.Normalize(vmin=-1, vmax=1)
```

即：

- 负相关偏冷色
- 正相关偏暖色
- 中间值接近白色

### 2. 为什么不再自定义中间色

之前尝试过：

- 整体加深
- 去掉白色中间过渡
- 设置最小白度

这些试验后来都被撤销。

当前最终保留的是：

- 原始 `RdBu_r`
- 只限制文字不要落入中间最浅区域

这是当前稳定版本，不建议轻易再改。

### 3. 色条口径

当前使用统一色条表示相关系数强弱，范围固定为：

$$
[-1, 1]
$$
